# Block Collection

> Source: https://www.aem.live/developer/block-collection

## Overview

Adobe Experience Manager's Block Collection comprises reusable components that serve as blueprints for blocks in your project. These blocks originate from production AEM projects and must demonstrate widespread adoption across multiple projects while maintaining sufficient abstraction for general reuse.

## Core Principles

The collection adheres to seven technical standards:

- **Intuitive:** Easy-to-author content structure
- **Useable:** No dependencies; compatible with boilerplate
- **Responsive:** Functions across all breakpoints
- **Context Aware:** Inherits CSS context automatically
- **Localizable:** No hardcoded content
- **Fast:** No performance degradation
- **SEO and A11y:** Search engine and accessibility friendly

## Boilerplate vs. Block Collection

**Boilerplate** contains the most frequently used blocks adopted by the majority of AEM projects. This curated set includes default content types and has open-source code available on GitHub at `adobe/aem-boilerplate`.

The broader **Block Collection** includes commonly-used blocks not meeting boilerplate threshold, requiring usage across more than half of all AEM projects. Its open-source code resides at `adobe/aem-block-collection` on GitHub.
